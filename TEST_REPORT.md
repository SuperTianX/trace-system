## 四、测试中发现并修复的问题

### 4.1 接口路径不匹配（3处）
| 后端 Controller 路径 | 前端 API 路径 | 修复方式 |
|---------------------|--------------|---------|
| `/api/v1/auth/role` | `/api/v1/role` | 改为 `/api/v1/role` |
| `/api/v1/auth/user` | `/api/v1/user` | 改为 `/api/v1/user` |
| `/api/v1/rule` | `/api/v1/rule-config` | 改为 `/api/v1/rule-config` |

### 4.2 缺少接口端点（4处）
| Controller | 缺少的端点 | 修复方式 |
|-----------|-----------|---------|
| RoleController | GET /page, PUT /{id}, DELETE /{id} | 新增 |
| UserController | GET /{id}, PUT /{id}, DELETE /{id} | 新增 |
| WorkReportController | GET /{id}, PUT /{id}/reject | 新增 |
| InventoryController | GET /{id} | 新增 |

### 4.3 接口参数不匹配（3处）
| 接口 | 问题 | 修复方式 |
|------|------|---------|
| ReportController | 路径与前端不匹配 | 更新 @GetMapping 路径 |
| ReconciliationController.assign | `@RequestParam` vs 前端JSON body | 改为 `@RequestBody` |
| ChainDiagnosisController.assign | `@RequestParam` vs 前端JSON body | 改为 `@RequestBody` |

### 4.4 实体字段缺失（2处）
| 实体 | 缺失字段 | 修复方式 |
|------|---------|---------|
| Role | description, status | 新增 + ALTER TABLE |
| User | email, phone | 新增 + ALTER TABLE |

### 4.5 安全配置（1处）
| 问题 | 修复 |
|------|------|
| `/api/v1/auth/refresh` 需要认证才能调用（循环依赖） | 加入 `permitAll()` |
