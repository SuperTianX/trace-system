**1.** ***\*概述\****

**1.1.** ***\*项目背景\****

钢铁行业生产、质量、仓储、销售全链路数据分散，炉、坯、卷、工单、质检、入库、出库、客户投诉之间缺乏统一管理，批次链路断裂、跨系统数据不一致、追溯困难、对账差异频发，导致质量异议难以定位、责任边界模糊、人工核对成本高、客户满意度低。

炉批卷追溯是钢铁 MES、质量管理、成本核算、质保书出具的核心基础能力，亟需建设一套覆盖全链路、可追溯、可对账、可诊断、可闭环的一体化系统，解决 “查不到、对不上、说不清” 三大痛点。

**1.2.** ***\*业务痛点\****

追溯链路断裂：炉、坯、卷、工单、质检、入库、出库关系不完整，质量异议无法定位根源。

跨系统数据不一致：MES 报工、质检判定、ERP 入库、库存台账数量长期差异，人工核对效率极低。

批次管理混乱：重复报工、未质检入库、MES 入库 ERP 未入库、批次断链、关联错误频发。

质量异议闭环缺失：异议登记、追溯、责任、整改、复核、归档缺乏统一平台，无法形成闭环。

责任边界模糊：出现质量问题或数据差异时，难以精准定位冶炼、轧制、质检、仓储、运输责任环节。

**1.3.** ***\*项目目标\****

**1.3.1.** ***\*项目目标\****

建立炉 - 坯 - 卷 - 工单 - 质检 - 入库 - 出库 - 异议全链路数据模型，实现批次唯一、关系可追溯、数据可校验。

支持正向追溯（炉→客户）、反向追溯（异议 / 卷→炉），一键定位全链路信息与异常节点。

自动执行报工 - 质检 - 入库 - 库存对账，识别差异、生成清单、支持责任分配与闭环处理。

智能诊断批次断链、质检缺失、未质检入库、重复报工、接口缺失、数量异常等问题并分级预警。

实现质量异议全流程闭环管理：登记→追溯→责任→措施→整改→复检→复核→归档。·

提供可视化链路图、统计报表、看板，支撑质量分析、生产复盘、管理决策。

**1.3.2.** ***\*适用范围\****

业务部门：质量管理部、生产车间、MES、仓储部、销售部、客户服务部。

·  业务场景：质量异议追溯、批次全链路查询、报工入库对账、库存批次管理、质保书数据支撑、生产异常复盘。

**2.** ***\*业务描述\****

**2.1.** ***\*核心业务实体\****

炉次：冶炼批次唯一标识，关联铸坯

铸坯：连铸成品，关联轧制批次

轧制批次：热轧 / 冷轧生产批次，关联卷号

卷号：成品钢卷唯一标识，关联质检、入库、出库

质检记录：成分、性能、尺寸、表面质量判定

报工记录：MES 工序产量、工时、操作人

入库出库：库存变动、仓库、数量、时间

质量异议：客户投诉、问题描述、处理结果。

**2.2.** ***\*核心业务流程\****

#### ***\*2.2.1 全链路主流程\****

冶炼（炉次）→连铸（铸坯）→轧制（批次）→成品（卷号）→质检判定→入库→出库→客户接收→质量异议

#### ***\*2.2.2 正向追溯流程\****

输入炉号 / 铸坯号 / 轧制批次 → 查询关联坯、卷、工单、质检、入库、出库、客户 → 生成可视化链路 → 高亮异常节点

#### ***\*2.2.3 反向追溯流程\****

输入投诉单号 / 卷号 / 出库单号 → 反向查询卷、批次、铸坯、炉次 → 定位工艺参数、生产异常、质检缺陷 → 生成追溯报告

#### ***\*2.2.4 报工入库对账流程\****

拉取 MES 报工、质检合格、ERP 入库、库存台账数据 → 按批次 / 卷号比对数量 → 识别差异 → 生成对账异常清单 → 责任部门确认 → 修复处理 → 复核关闭

#### ***\*2.2.5 断链诊断流程\****

定时校验链路完整性 → 识别缺环节、关联错误、数量异常、状态冲突 → 标记断链并分级 → 影响范围分析 → 责任分配 → 修复 → 复核关闭

#### ***\*2.2.6 质量异议闭环流程\****

异议登记 → 自动批次追溯 → 责任初步判定 → 处理措施制定 → 整改执行 → 复检复核 → 客户确认 → 关闭归档

### ***\*3.4 轧制批次管理\****

· 批次号、关联铸坯号、产线、工单编号、轧制日期、工艺参数

· 批次合并、拆分、异常标记、导出

### ***\*3.5 卷号档案管理\****

· 卷号、规格、重量、材质、质量等级、关联轧制批次

· 入库单号、库存位置、库存状态、出库单号、客户信息

· 全生命周期状态跟踪、批量操作、导出

### ***\*3.6 质检记录管理\****

· 关联类型（炉 / 坯 / 卷）、关联编号、检验项目、检验值、判定结果

· 检验时间、检验人、复检记录、改判记录、不合格原因、处置方式

· 新增、修改、查询、导出、审批

### ***\*3.7 报工管理\****

· 工单编号、工序、批次号、卷号、报工数量、报工时间

· 操作人、班组、设备、异常说明、撤销记录、审核状态

· 录入、修改、撤销、批量导入、导出

### ***\*3.8 入库出库管理\****

· 单据编号、类型（入库 / 出库）、批次号、卷号、数量、时间

· 仓库、操作人、关联工单、关联质检、客户、运输信息、状态

· 录入、审核、查询、导出、批量操作

### ***\*3.9 正向追溯\****

· 输入：炉号 / 铸坯号 / 轧制批次号

· 输出：完整链路树（炉→坯→卷→质检→入库→出库→客户）

· 展示：关键参数、时间、责任人、异常节点高亮、详情查看、导出

### ***\*3.10 反向追溯\****

· 输入：投诉单号 / 卷号 / 出库单号

· 输出：反向链路（异议→卷→批次→坯→炉）

· 展示：工艺参数、生产异常、质检缺陷、责任环节、追溯报告、导出

### ***\*3.11 报工入库对账\****

· 数据来源：MES 报工、质检合格、ERP 入库、库存台账

· 比对维度：批次、卷号、数量、时间

### ***\*3.12 断链诊断\****

### ***\*3.13 质量异议处理\****

**4.** ***\*非功能需求\****

**4.1.** ***\*性能要求\****

Ø 常规的系统业务页面刷新时间＜2s,报表生成刷新时间＜5s，保证用户使用的体验性；

Ø 单记录录入/修改的响应时间小于3秒；

Ø 系统稳定运行率≥99.8%

Ø 稳定运行率的定义：Y＝T1/(T1+T2) × 100%，备注：测试时间＝T1＋T2；T1：有效运行时间，T2：故障处理时间。考核时间为验收前连续720小时；

Ø 数据接口响应时长最长不大于60秒；

Ø 各模块之间可增加跳转功能，后续根据用户实际使用要求增加，生产人员也可通过首页设置一键跳转到对应界面查看，减少不必要操作；

**4.2.** ***\*系统的安全性\****

通过用户使用授权管理，系统操作分配用户权限，用户按照不同的角色职责化分为不同的角色群组，功能模块按照不同的业务类型分为业务群组，不同的角色群组授予不同的业务群组权限，实现功能权限、数据权限的分配，确保数据操作的安全性。

**4.3.** ***\*系统的可靠性\****

数据校验是增强系统可用性和可靠性的重要手段，系统要求对输入的数据进行有效性、合法性校验，数据校验可以排除许多数据不一致的现象，从逻辑上、数据完整性上保证数据的质量，做到“正确的输入有正确的结果，错误的输入有正确的响应”。 系统应具有较强的容错性，对于用户的误操作，应给予友好的提示；对于系统出现的异常，应向用户解释原因，提示用户如何处理；对于已经发生错误或异常，系统应尽可能恢复到原来操作状态。

数据备份：允许用户进行数据的备份和恢复，以弥补数据的破坏和丢失。 

记录日志：本系统应该能够记录系统运行时所发生的所有错误，包括本机错误和网络错误。这些错误记录便于查找错误的原因。日志同时记录用户的关键性操作信息。 

提供7*24小时服务。
